✅ IMPLEMENTADAS LAS SIGUIENTES CARACTERÍSTICAS:

1. SISTEMA DE ADMIN PRIVILEGIADO
   - El número 7571040521 tiene privilegios especiales de admin
   - Puede ejecutar comandos de admin en cualquier grupo donde el bot sea admin
   - No necesita ser admin del grupo para ejecutar comandos privilegiados
   - Archivo de datos: mutedusers.json

2. COMANDO .MUTE
   - Sintaxis: .mute @usuario 5m (razón)
   - O responder a un usuario con: .mute 5m razón
   - Duración: se especifica con m (minutos), h (horas), d (días)
   - El usuario muteado no puede usar comandos durante ese tiempo
   - Cuando intenta usar comando, recibe notificación de mute con tiempo restante

3. SISTEMA .STOCK MEJORADO
   - Nuevo comando: .setstock (texto del stock) + imagen opcional
   - La imagen se puede agregar respondiendo a una imagen con .setstock
   - O enviando una imagen + .setstock en el mismo mensaje
   - Comando .stock: Muestra todos los stocks o uno específico
   - Si existe imagen guardada, se envía con la información
   - Archivo de datos: stockdata.json

4. SISTEMA .PAGO
   - Nuevo comando: .setpago (texto del método) + imagen opcional
   - Funciona igual que .stock pero para métodos de pago
   - Comando .pago: Muestra el método de pago configurado con imagen
   - Archivo de datos: pagodata.json

5. COMANDO .N MEJORADO
   - Ahora soporta:
     • Imágenes
     • Stickers
     • Audios
   - Puede responder a mensaje con media y enviar a todos
   - Mantiene el sistema de mención fantasma (mention al todos sin escribir @)

6. COMANDO .STIKER
   - Convierte imagen/video a sticker
   - Sintaxis: Responder a imagen con .stiker
   - O enviar imagen + .stiker en el mismo mensaje
   - Igual que .s pero con nombre diferente

7. SISTEMA DE COMANDOS PERSONALIZADOS
   - Crear comando: .create nombrecomando
   - Ejemplo: .create diamantes
   - Establecer contenido: .setdiamantes info del producto
   - O con imagen: Responder imagen con .setdiamantes
   - Usar: .diamantes (mostrará lo guardado)
   - Los comandos personalizados se guardan en JSON
   - Archivo de datos: customcommands.json
